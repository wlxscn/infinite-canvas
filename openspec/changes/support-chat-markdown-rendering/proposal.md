## Why

当前右侧聊天面板将消息正文按纯文本渲染，只保留换行，无法正确展示设计助理常见的结构化回复，例如列表、代码块、引用和表格。随着聊天面板逐步承担更强的设计说明、分析对比和执行建议职责，继续停留在纯文本展示会让信息密度高的回复难以阅读，尤其在助理返回步骤分解、参数对照或样式对比表时体验明显下降。

现在推进这项变更的价值在于，先把聊天消息的“最终态阅读体验”补齐，同时保持当前已经调顺的流式打字机效果不被 Markdown 中间态破坏。这样可以在不改动服务端消息协议的前提下，让设计助理回复更接近用户对 ChatGPT 式对话体验的预期。

## What Changes

- 将右侧聊天面板中的助理最终消息从纯文本显示升级为 Markdown 渲染，支持常见段落、列表、强调、引用、链接、行内代码与代码块。
- 为助理最终消息启用 GFM 扩展，明确支持表格展示，并在窄侧栏中为表格提供局部横向滚动容器，避免撑破聊天气泡或整个侧栏布局。
- 保持当前流式回复阶段继续使用纯文本打字机效果；仅在消息完成后切换到 Markdown 最终态渲染，避免未闭合 Markdown 语法在流式过程中造成闪烁和结构跳变。
- 维持现有消息数据结构和服务端 `/chat` 文本输出协议，不引入 HTML 直通渲染，也不要求服务端返回新的富文本结构。
- 明确本次变更的非目标：
  - 不在流式阶段实时解析和渲染 Markdown
  - 不支持原始 HTML 透传
  - 不扩展为通用富文本编辑器或可交互文档查看器

### Modified Capabilities
- `ai-design-canvas`: 调整右侧助理聊天消息的展示要求，使助理最终消息支持 Markdown/GFM 渲染和表格展示，同时保持流式阶段的纯文本打字机体验。

## Impact

- 主要影响 `apps/web/src/features/chat/components/AgentSidebar.tsx` 中的消息正文渲染路径，以及与流式打字机衔接相关的消息显示逻辑。
- 会影响 `apps/web/src/index.css` 中聊天消息正文、代码块、列表、引用、链接和表格的样式体系，并需要为侧栏窄宽度场景补充横向滚动与溢出控制规则。
- 预计会新增前端 Markdown 渲染依赖（如 `react-markdown` 与 `remark-gfm`），但不要求修改 `packages/shared` 中的消息类型或 `apps/agent-api` 的返回结构。
- 需要补充或更新聊天侧栏的单元测试，覆盖 Markdown 最终态渲染、表格可见性以及流式纯文本到最终 Markdown 的切换行为。
